From URL:

URL of JSON:

You must type an URL of your theme's .json file

There are strict rules for the structure of the remote directory.

The following must be true for the JSON at the URL, in addition to the rules found in THEMES.md:

  • Must have a name key, with the value being the shortname of the extension. Same rules apply to the shortname, as does for javascript variable names.
  • Must have a css key, with a value of the absolute URL of the CSS for the theme.
  • Must have a bg key, with a value of the absolute URL of the background image.
  • If an icon is wanted, must have an icon key, with a value of the absolute URL of the icon image.

In addition to this, the server the JSON is hosted on must allow cross-origin GET requests from all domains (i.e. return the Access-Control-Allow-Origin: * and, optionally, Access-Control-Allow-Methods: GET headers).

Background, icon and CSS are hotlinked.

Themes are automatically updated once a day.

Changelog

v0.8.2.2 released on april 31st, 2015

  • New:
  • Brought back extra match info caching, this time however the cached information will last 5 minutes before requesting fresh information again
  • Rarity is now also shown near market price for Dota2 items
  • NEW LOGO!!!!!!11 by /u/farblood
  • Item groups now differentiate between identical items
  • Added options to change small/medium/large bet %
  • Added option to select for which inventory groups you want inventory statistics to be calculated
  • Added button in options to reset your user settings to default ones
  • Bug fixes:
  • 'undefined' item name when doing a CSGOLounge item value check if you were using non-English language for Steam
  • Fixed match notifications not working thanks to a bugfix that degraded it even further
  • Fixed rare cases where market price didn't fetch because CSGOLounge added items that have unnecessary spaces at the end of item name
  • Changed Canadian dollar symbol to be shorter
  • Finally fixed the inconsistency in detecting when betting inventory has loaded
  • Changed market price loading function to not load market prices pointlessly for items that are already market priced
  • Fixed 'Total value bet' calculated incorrectly if you were using non-USD currency
  • Auto-betting/-returning now works in incognito mode
  • Fixed broken inventory loading if the user used 'Force refresh' button
  • Item groups now work with our cached inventories
  • Fixed 'Base' quality being ordered too high in inventory statistics
  • Fixed a rare case where trade offer was not automatically accepted because of faulty check for how much time is left for trade offer to accept
  • Minimum auto trade offer accepting delay is now forced properly to 10 seconds (this also caused auto offer accepting to not work)
  • Accounted for chrome.alarms bug when setting system time back which caused scheduled tasks to be messed up, resulting in things like auto-bumping being broken
